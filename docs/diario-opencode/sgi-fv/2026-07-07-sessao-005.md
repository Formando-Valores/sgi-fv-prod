# Sessão 005 — 2026-07-07 (parte 2)

## O que foi feito

### 1. Correção de escopo de organização
- Erro "Não foi possível validar o escopo da organização" ocorria ao acessar dashboard
- Causa: `resolveOrganizationScope()` consultava tabela `organization_members` (inexistente) em vez de `org_members`
- Fix: alterado `.from('organization_members')` para `.from('org_members')` em `ProcessesSection.tsx`

### 2. Conversão de BRL (R$) para EUR (€)
- Adicionada função `formatEuro()` e constante `EUR_RATE = 6` em `src/lib/servicesCatalog.ts`
- ~48 ocorrências de "R$" substituídas por `formatEuro()` em 7 arquivos:
  - `ProcessesSection.tsx`, `ProcessList.tsx`, `AdminDashboard.tsx`
  - `ProcessDetails.tsx`, `ServicesSection.tsx`, `ReportsPage.tsx`
  - `src/lib/reports.ts`
- Labels "(R$)" alterados para "(€)"
- Usado `toLocaleString('pt-PT', { style: 'currency', currency: 'EUR' })` para formatação nativa

### 3. Simplificação do dashboard do cliente
- Dashboard do cliente estava poluído com gráficos administrativos (donuts, barras, tempo médio) e tabela de processos
- `DashboardSection.tsx`: quando `isClientScope = true`, renderiza APENAS o `ClientProcessProgressPanel` (jornada do cliente com steps + histórico)
- Highlights cards, gráficos e tabela permanecem apenas para admins
